Searching and Sorting

 

Searching

Searching in Java is the process of finding a specific object within an Array or ArrayList. There are two main searching algorithms that are used in Java.

 

Sequential Search

Sequential search is a searching algorithm that scans each index of an Array or ArrayList to find a specific element. Where n = array.length, the average number of comparisons is n/2. In the worst case, n comparisons are needed to find the element or to make certain that the element in not within the array. Therefore, the time complexity of this array is O(n).




The sequential search algorithm scans each index to find the element. (Photo: My Assignment Help)

 

Binary Search

Binary search is another searching algorithm. In order to use binary search, the elements of the Array or ArrayList must be arranged in ascending or descending order. This search compares the target value with the middle element of the remaining search range. It is a faster "divide and conquer" algorithm that only needs 20 comparisons to search for an element in an array of 1,000,000 elements. The time complexity of binary search is O(log n).




The binary search algorithm splits the array in half each time to find the element. (Photo: Java67)

Sorting

Oftentimes, the elements of an array need to be sorted in ascending or descending order. For example, you might need to sort people by height, name, weight, etc. In the following sections, you will see four of the most commonly used sorting algorithms.

 

Selection Sort

Selection sort is a sorting algorithm that selects the greatest (or smallest) value in an Array or ArrayList and swaps it with the first last element in the array. It repeats this process until the array is completed sorted in ascending or descending order. It is a consistent algorithm that always makes n(n-1)/2 comparisons and has a time complexity of O(n^2).

The selection sort algorithm swaps the largest or smallest value in an array with the first or last element in the array to sort it. (Photo: Stack Overflow)

 

Insertion Sort

Insertion sort is another sorting algorithm that divides an Array or ArrayList into two sections: a sorted array and an unsorted array. The first value in the unsorted array is then compared with each element in the sorted array until its correct position is found. This process is then repeated until the entire array is sorted. When the array is already sorted, insertion sort will take n-1 comparisons. When the array is sorted in reverse order, it will take n(n-1)/2 comparisons. On average however, insertion sort is an O(n^2) algorithm.




The insertion sort algorithm splits an array into a sorted and unsorted section to sort the array. (Photo: Geeks for Geeks)

 

Merge Sort

Merge sort is a fast sorting algorithm that splits an Array or ArrayList in half. It then repeats this process, splitting each smaller array into half until the each array contains only one element. Finally, it merges together all of the arrays in the correct order until one final, sorted array is created. Merge sort has a time complexity of O(log n).




The merge sort algorithm splits an array multiple times to sort the array. (Photo: Geeks for Geeks)

 

Quicksort

Quicksort is another fast sorting algorithm. Unlike mergesort, it selects a "pivot" to split the array. All elements to the left of the pivot are made to be <= pivot and all elements to the right of the pivot are made to be >= pivot. It then repeats this process until the entire array is sorted.




The quicksort algorithm uses a pivot to sort an array. (Photo: Techie Delight)
